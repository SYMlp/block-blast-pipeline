# 认知推导链:从"游戏本质"到"DSL 验证层该测什么"

> 综合四轴认知推导(玩的本质 / 动机心理 / 休闲特化 / 感受→指标桥)+ 真实 headless sim 数据的收口。
> 用途:给"游戏变体 DSL"的**意图层 + 验证层**一个从根事实推导出来的指标依据,而不是拍脑袋选指标。
> 这条链本身就是 demo 的差异化叙事:**指标不是猜的,是从玩的本质推导、再被真实 sim 数据淘汰了弱代理(占用率)而留下的。**

---

## 一、推导链(根 → 休闲特化 → 可测边界)

### 根:人为什么玩、乐趣从哪来

- **游戏 = 自愿走进一个隔离的规则空间,用不必要的障碍把自己围住**(Huizinga 魔法圈 / Suits)。
- **乐趣(fun) = 大脑成功掌握"模式"时的内啡肽奖励**(Koster《A Theory of Fun》)。无聊 = 模式被吃透。
- **人玩游戏在买三样:胜任感 / 自主感 / 联结感**(SDT)。让人停不下来的是"比预期好一点"的多巴胺奖励预测误差(Schultz)+ 可变比例强化(Skinner)+ near-miss/Zeigarnik 钩子。
- **心流 = 胜任感的时间序列形态**:挑战 ≈ 技能且略高。
- **方块消除是近乎纯粹的 pattern-mastery 引擎**:MDA Challenge(主)+ Sensation/juice(强辅)+ 一丝 Submission;结构上 Agôn + Ludus + 微量 Alea(发牌手气软化失败归因)。乐趣维度极少极纯 → 是"感受→指标"推导的绝佳样本。

### 拱顶石:两道悬崖

留存流失有**两种相反成因**:

| | 无聊崖 | 挫败崖 |
|---|---|---|
| 成因 | 模式被吃透 | 模式装不起 |
| 行为信号 | 会话渐短、决策变快、不再追分 | 早夭局多、同难度反复失败、关闭而非重开 |

**难度曲线 = 把玩家维持在这两崖之间的窄带。** 这是 DSL 难度维度的根目标:不是"越难越好"或"越易越好",是**双边带内**。

### 休闲特化:把体验重新打包

- **休闲 = 把"胜任 + 秩序 + 情绪调节"从"高门槛长周期"重打包成零摩擦、随时取放的微剂量。优化的是"触达体验的摩擦",不是体验高度。**
- **四条护栏**(跌破即背叛休闲本质):① 低摩擦→开局即有可玩步 ② 可中断→单局短且自包含 ③ 秩序/完成→消除频繁 ④ 情绪稳压→失败宽容+即时反馈。

### 可测边界:哪些 headless 能测、哪些必须真人

- **headless 能测 = "胜任感 + 心流"族**——它们是**游戏规则与状态空间的客观属性**,会玩的 bot 跑足够多局就能统计。
- **必须真玩家 = 自主感、联结感、真上头、次留/7留、付费、juice 主观冲击、认知负荷、难度的绝对人类感知**——这些是人的反应,不是游戏属性。
- **两个诚实点**:① Goodhart 陷阱——代理≠真感受,别让指标变成目标本身。② AI agent 测难度**只能相对排序不能绝对标定**(agent 系统性比人菜,相关可能来自"一致地菜")。
- **流水线架构(两份调研自发收敛)**:**变体先过 headless 自动门(淘汰死局/太易/太难/无惊喜)→ 幸存者才交真人测留存付费。** headless 是第一道粗筛闸,不冒充全能。

## 二、总表:感受 → 代理 → 怎么算 → 可信度(× 真实 sim 数据交叉验证)

> sim 数据来自 200 局/变体 greedy agent,见 [`harness/sim/last-run.json`](../harness/sim/last-run.json)。

| 人类构念 | 代理指标 | headless 怎么算 | 实测/可信度 |
|---|---|---|---|
| **挫败崖(公平/不劝退)** | 单局过短比例、早夭 | stepsMedian / P90 偏低 | **强**:hard-mode 中位 2 步=贴挫败崖 |
| **无聊崖** | 会话长而无脑、低消除、低惊喜 | clearRate 低 + 奖励熵低 + 步数长 | **强**:relaxed clearRate 0.254 + 步170=偏无聊端 |
| **公平感(RNG 不坑)** | 强制死局率 / 安全网救场率 | 记 refillTray 重试次数 | **强效度**:已实现(rescueRate) |
| **秩序/完成节拍** | 消除频率 | clearRate | **中-强**:已有 |
| **胜任感空间** | greedy vs random 得分/存活差 | 跑两种 agent 对比(可扩) | 中-强 |
| **惊喜/多巴胺** | 奖励事件方差/熵 | clear gain 分布的熵 | 中:已实现(rewardEntropy) |
| 心流位置 | gameOver 占用率分布 | filled/格数 | **弱**:实测平(0.59–0.66),被千篇一律的 greedy agent 抹平,**淘汰** |
| 爽快感(主观) | combo 规模(只测给没给机会) | maxCombo 分布 | 弱效度:greedy 不 chain |
| 上头/留存/付费/自主/联结 | —— | **测不了** | 必须真人 A/B |

## 三、结论:DSL 验证层的锚

**用"双边带"而非"单向最大化"做验证**。每个变体的验证 = 一组"代理指标必须落在 [地板, 天花板] 带内",跌出 = 这个变体的假设失败(对应休闲游戏 A/B ~97% 的失败率)。

落地的 v1 锚(强效度优先):
1. **stepsP90 / stepsMedian**(挫败崖 + 可中断性)
2. **clearRate**(无聊崖 + 秩序节拍)
3. **rescueRate**(公平感)— 调研里列为"最该秀的一条",已实现
4. **rewardEntropy**(惊喜)— 已实现

被数据淘汰:**gameOver 占用率(avgEndOccupancy)**——文献对、但需要会变的玩家,greedy agent 把它抹平了。指标不是猜的,是被数据筛出来的。
